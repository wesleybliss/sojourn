# Plan to Fix Firebase Module Not Found Error in Vercel Deployment

## Problem
The Vercel-deployed API function fails with:
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'firebase' imported from /var/task/api/index.js
```
This occurs because the Firebase package is not found at runtime.

## Root Cause
1. The `apps/api/build.js` script uses esbuild to bundle the API function.
2. It externalizes certain major dependencies (including `firebase` and `firebase-admin`) to avoid bundling large SDKs.
3. Externalized dependencies are expected to be present in the runtime `node_modules`.
4. Although `firebase` and `firebase-admin` are dependencies of the `@repo/shared` workspace, they are not automatically available in the API workspace's node_modules during Vercel deployment because the build process only includes the API workspace's direct dependencies.
5. Consequently, at runtime, the Node.js module resolver cannot locate `firebase`.

## Solution Implemented
Modified the build script to bundle Firebase instead of externalizing it, ensuring the Firebase code is included in the output bundle.

### Changes Made
1. **Edited `apps/api/build.js`**:
   - Located the `majorDependencies` array inside the `pluginPathAliases` function (lines 142-153).
   - Removed `'firebase'` and `'firebase-admin'` from this array.
   - This causes esbuild to bundle these packages instead of marking them as external.

## Verification
1. **Build Success**: 
   - `pnpm --filter api build` completed successfully
   - Firebase and firebase-admin appeared in the "Found dependencies to bundle" list

2. **Linting Passed**:
   - `pnpm lint` completed with no warnings or errors across all workspace projects

3. **Bundle Inspection**:
   - Confirmed Firebase code is included in `/home/wes/localhost/sojourn/api/index.js`
   - Found Firebase util references (line 42654+) 
   - Found Firebase app initialization code (line 115287)
   - Verified Firebase imports are properly transformed by esbuild

4. **Bundle Size**: 5.3MB - reasonable for a serverless function with Firebase

## Result
The Firebase module not found error should now be resolved in Vercel deployments because:
- Firebase code is bundled directly into the API function output
- No longer relies on external node_modules resolution in the Vercel environment
- All required Firebase functionality is self-contained in the bundle
